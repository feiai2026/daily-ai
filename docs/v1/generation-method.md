# Daily AI v1 生成方法

## 1. 目标产物

每天生成：

- `reports/v1/YYYY-MM-DD-candidates.json`：公开 API 返回的候选内容。
- `reports/v1/YYYY-MM-DD.md`：最终中文日报。

日报必须使用：

- `templates/v1/daily-report-template.md`

## 2. 信源

### GitHub / 开源

自动采集 GitHub Search API，关键词和阈值在 `config/v1/sources.json`。

默认查询数量控制在未认证免费限额内。频繁运行时可通过环境变量 `GITHUB_TOKEN` 提高 API 限额，但不是必需条件。

重点关注两类项目：

1. 成熟且仍活跃的项目：
   - 总 stars 达到配置阈值。
   - 最近 14 天有提交。
2. 新出现且增长较快的项目：
   - 最近 7 天创建。
   - stars 达到新项目阈值。

每日报告最多选择 5 个项目。避免仅按总 stars 排列一批长期热门旧项目。

### Hacker News

自动使用 HN Algolia API 搜索最近 3 天的 AI 相关帖子。

满足任一条件即可进入候选：

- points 大于等于 150。
- comments 大于等于 80。

每日报告最多选择 5 条讨论。

### 官方公司博客

检查最近 7 天发布的内容：

- 模型、API、产品
- 研究与评测
- 开发者工具
- 企业部署
- 安全与治理

官方重大更新可进入“今日必读”，普通更新进入“公司动态”。

### 独立作者

检查最近 14 天发布的 AI、LLM、Agent、评测、模型架构和开发者工具内容。

如果内容最初来自 X，必须找到公开可访问的原始链接、作者博客、HN 讨论或可信汇总页。

## 3. 分层筛选

### A 级：今日必读候选

满足任一：

- 官方重大发布。
- HN points 大于等于 300。
- HN comments 大于等于 150。
- 新 GitHub 项目在 7 天内达到 500 stars。
- 成熟 GitHub 项目近期出现重要 release 或功能变化。
- 核心作者发布并被多个公开来源讨论的重要内容。

### B 级：分区收录

满足任一：

- HN points 大于等于 150。
- HN comments 大于等于 80。
- 新 GitHub 项目在 7 天内达到 100 stars。
- 总 stars 大于等于 1000，且最近 14 天有更新。
- 官方博客最近 7 天的相关更新。

### C 级：只保留在候选数据

- 来源可靠但热度较低。
- 可能后续发酵。
- 同一事件的二级报道。
- 与日报主题相关性较弱。

## 4. 去重与来源优先级

同一个事件只保留一个主条目。

来源优先级：

1. 官方原文
2. 作者原文
3. GitHub 仓库或 release
4. Hacker News 讨论
5. Techmeme 等汇总页
6. 其他二手来源

HN 讨论有独立价值时，可以作为同一条目的补充链接。

## 5. 写作规则

- 默认输出中文。
- 英文标题翻译为中文，并保留原题。
- 先写事实，再写判断。
- 不把“很火”当作“很重要”。
- 每条重点内容必须解释“为什么值得看”。
- 区分来源声称、媒体报道和可验证数据。
- 有争议的事件应使用中性表述，并同时提供原文与讨论链接。

每条“今日必读”包含：

- 中文标题
- 英文原题
- 来源
- 日期
- 量化信号
- 链接
- 为什么值得看
- 关键信号

GitHub 条目包含：

- 仓库链接
- stars、forks
- 创建时间或最近更新时间
- 项目简介
- 为什么值得看

HN 条目包含：

- points、comments
- 原始文章链接
- HN 讨论链接
- 讨论焦点

## 6. 固定执行步骤

1. 运行采集脚本：

   ```bash
   node scripts/v1/collect-v1.mjs
   ```

2. 打开当天候选 JSON。
3. 检查官方公司博客。
4. 检查独立作者来源。
5. 去重并按 A/B/C 分层。
6. 按模板生成中文日报。
7. 保存到 `reports/v1/YYYY-MM-DD.md`。
8. 运行：

   ```bash
   node scripts/v1/check-report.mjs reports/v1/YYYY-MM-DD.md
   ```

## 7. v1 边界

v1 不做：

- 付费 X API
- 依赖登录态的抓取
- 全网覆盖
- 复杂主观评分
- 自动发布到第三方平台
- 内置定时任务

v1 要做到：

- 免费公开源
- 客观指标优先
- 原始链接完整
- 中文结构化日报
- 可由不同 AI Agent 重复执行
